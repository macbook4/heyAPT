d# heyAPT MVP 실행 태스크 보드

## 운영 원칙
- 기준 문서: `docs/plan-spec.md`
- 목표: 4주 내 `등록 POI 수` 최대화 + `주소 오표기율 <= 5%`
- 범위: 수도권, 일반회원 등록, 승인 후 게시(영업일 48시간 SLA)

## W1 (기반 구축)
### 목표
- Web + App(WebView) 동시 개발 가능한 최소 구조 완성
- 인증/권한/DB 기본선 확정

### 핵심 태스크
- `P0` 모노레포 구조 생성: `apps/web`, `apps/mobile`, `packages/bridge-contracts`
- `P0` Supabase 프로젝트 연결 및 환경변수 체계 확정
- `P0` 로그인(소셜+이메일) 플로우 구축
- `P0` `pois`, `profiles`, `audit_logs` 스키마 작성
- `P0` RLS 정책 적용(기준: `docs/rls.sql`)
- `P1` Naver Map 초기 화면 + bounds 이벤트 핸들링

## W2 (등록/검수 플로우)
### 목표
- 일반회원 POI 등록과 운영 승인 파이프라인 완성

### 핵심 태스크
- `P0` POI 등록 폼: 주소 검색, 가격, 거래유형, 사진 1장+, 상세설명
- `P0` 사진 업로드(권리 보유 동의 필수 체크)
- `P0` 등록 상태머신: `draft -> pending_review -> published/rejected`
- `P0` 운영자 검수 화면(목록/상세/승인/반려)
- `P1` 반려 사유 템플릿 및 재제출 플로우
- `P1` 중복 수작업 처리 UX(병합/반려 기준)

## W3 (탐색/품질/관측)
### 목표
- 지도 탐색 경험과 운영 품질 지표 안정화

### 핵심 태스크
- `P0` `GET /api/pois` bounds 기반 조회 최적화
- `P0` POI 상세 패널(회원/주소/사진) 완성
- `P0` 주소 오표기 신고 기능 + 운영 재검수 큐
- `P0` 감사로그 기록(`request_id`, `actor_user_id`, before/after)
- `P1` 대시보드: 등록 POI 수, 검수 SLA, 주소 오표기율
- `P1` 에러 모니터링/알림 설정

## W4 (하이브리드 앱 마감/출시 준비)
### 목표
- 웹/앱 동시 출시 가능한 안정성 확보

### 핵심 태스크
- `P0` Capacitor iOS/Android 빌드 검증
- `P0` 로그인 세션 유지/만료 처리 일관성 검증
- `P0` 릴리즈 체크리스트(보안키, 도메인 allowlist, cap sync)
- `P0` 운영 런북(신고 대응, 반려 기준, 장애 대응)
- `P1` 성능 튜닝(지도 이동 시 재조회 체감 개선)
- `P1` 출시 후 1주 관측 계획(지표 리포트 템플릿)

## 역할별 담당
- Product/운영
  - 검수 정책, 반려 기준, SLA 모니터링, 품질 리포트
- Frontend(Web)
  - 지도/등록폼/검수UI/상세패널/에러 UX
- Mobile(Wrapper)
  - WebView 래퍼, 세션 브릿지, 앱 배포 설정
- Backend
  - API, RLS, 감사로그, 파일 업로드 정책
- QA
  - 권한/검수/주소오표기/앱세션 회귀 테스트

## Definition of Done (MVP)
- 수도권에서 회원이 POI 등록 후 48시간 내 승인/반려 가능
- 게시된 POI가 지도와 상세 패널에서 정상 노출
- 권한 모델(`guest/owner/admin`) 위반 케이스 차단
- 주소 오표기율 5% 이하 측정 가능(신고+검수 로그 기반)
- iOS/Android WebView 앱에서 핵심 플로우 동작

## 리스크 및 대응
- 등록량 급증으로 검수 병목
  - 대응: 우선순위 큐(신규/신고/재제출) 분리
- 주소 데이터 불일치
  - 대응: 검수 단계에서 좌표-주소 재확인 강제
- 사진 권리 분쟁
  - 대응: 업로드 동의 로그 + 즉시 블라인드/재검수 절차
